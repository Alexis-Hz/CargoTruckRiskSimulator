Cargo Simulator Manual and Documentation

About the Cargo Simulator

The cargo trip simulator is an event driven simulation that simulates cargo trips sent from different departure points, and arrive at different ports of entry.

In our specific implementation we used maquiladoras (part assembly factories) in Cd. Juarez, Mexico as departure points, or warehouses to these maquiladoras in some cases.

We used the 4 international bridges between Cd. Juarez and El Paso, TX. Although we used these POEs (Points of Entry) and departure points, the application can be easily used for any kind of port of entry and any kind of departure point. That said, you would have to adapt the code for custom databases.

Quick Overview

The cargo trip simulator takes as input an event scenario created in the Event Manager application. It then fetches all the necessary data as dictated by the Event Simulator from the Cargo Database, then updates the appropriate entries in the cargo database to reflect the state of the simulation and in the case of the webapp front end sends all the pertinent data to the cargo portal to be displayed. The Simulator may also take input from the webapp front end. In the case of the command line version of the Cargo Simulator the input and output are routed through the command line interface.

How is this documentation going to work?

This manual is inbedded in the code. This means that it is supposed to be located inside the actual eclipse project_folder/doc folder. The documentation consists of three main components:

  • The typed manual (The part you are actually reading right now), which goes over the simulator explaining the high level features and interactions between packages and classes.
  • The auto-generated Javadoc documentation, which is generated from the javadoc comments all thorough the code. This Html based documentation is automatically generated using the Ant build.xml file. This is covered here: Ant Build File and only builds Javadoc html for the sim package, not the ui or the tds packages.
  • The class diagram for the system. Parts of the class diagrams will be inserted in the manual in a on-need basis. Further description of the class diagram can be found here: Class Diagram Overview. The diagram itself can be found in the root root folder for the documentation dhs_version/doc. The diagram was created using Dia, a free diagram created application, but a .jpg version of the diagram should be next to the .dia version if you wish to avoid downloading dia

The three components of the documentation are used to create this manual and will be referenced whenever needed

Local Data for this project

In order for this project to run the data from the GIS servers had to be imported locally. This information is located in the JSP folder. This data includes the kml files for the routes as well as the scenario file that the simulator reads.

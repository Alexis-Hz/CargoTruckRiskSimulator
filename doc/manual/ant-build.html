Ant Build File Overview

The ant build xml file has a set of targets to simplify the compiling, management and deployment of the Cargo Simulator. We will briefly explain the use of each target here.

build

Depends on compile, simply calls compile. If there is need to add extra code and compile it, add the compile target for the extra code here

clean

Deletes the build directory containing the binaries

compile

Depends on clean, compiles all the source code

javadoc

Creates all the Javadoc Html from the jadoc comments in the code and stores it in the doc folder

runj

Depends on compile, Runs the ZenTerminal, this target is for the use of the jsp front end, use runt instead when wanting to run the Cargo Simulator

runt

Depends on compile, Runds the SimControlTerminal, call this target when wanting to run the application from secure shell or the terminal

runu

Not Used, runs SimControlView

warit

Depends on build, creates the war for the Cargo Simulator file and saves it in the root folder of the project.
